2020/05/25

무작정 정적블로그를 갖고 싶다! 내 이름을 가진 서버형 블로그를 갖고 싶어 졌다! 싶어서 제대로 알아보지도 않고, 구글링도 제대로 안하고 내 마음대로 손가는 대로 생각 나는대로 Hexo + Git 블로그를 열어 버린지 2주가 되었다. 뭔가 해보려고 계속 이것도 건드리고 저것도 건드리고 해서 오류에 오류 범벅으로 뒤덮혀있는 블로그를 얼추 사람 구실 하게끔 기본적인 세팅을 드디어 한 것 같다.

그간 한글깨짐 때문에 고민의 고민을 하던 도중 오늘은 해결해 봐야지 하면서 시작한 구글링. 저어어언혀 md파일 한글 깨짐에 대한 솔루션을 주는 포스트가 없

md파일 저장할때 txt메모장으로 열어서 저장하면 글자가 깨진다는것을 드디어 알 아 차 려 버렸다.

그간 한글 깨지는 이유가 뭔지 진짜 삽질하면서 찾아 봤는데…(코딩상의 utf-8이 잘못 된 것인지 웹폰트 호환이 잘 안맞는 것인지..

이것저것 진짜 있는거 없는거 다 찾아 보기 까지 하고) 혹시나 인코딩 문제인가해서 .js도 열어보고 인코더, 인코딩의 문제가 있는지 까지 config의 config 까지 다 뒤져가면서 겨우겨우겨우겨우겨우 봤더니 고작 ? 그냥 메모장 때문 이였다는것이 충격적이였다.

코딩을 하다보면 비슷한 경우를 종종 찾아 볼 수 있다. 간단하고 당연시 여겼던 것에서 오류가 검출 되는 경우가 잦다. 요즈음의 진화된 언어들은 컴파일러가 알아서 잡아주지만, C나 어셈블리어 같은 원초적인 친구들은 오류가 있다는 것만 뱉어 낼뿐 (있다고 말도 안해주고 그냥 뻗어 버리는 친구들도 많다.) 불친절한 녀석들이 많다.(그래서 C언어와 모듈만 다루는 나는 좋은 컴파일러를 쓰시는 분들이 조금 부럽기도 하다.)

아무튼 결론적으로 말하자면 어렵다고 생각했던 문제도 사실 심플한 해답을 가지고 있을 수 도 있다는 생각을 가지고, 포기하지 말고 츄라이 해봐야 한다는 것 을 다시 한번 느끼면서 말을 마치려 한다.
